# Результаты нагрузочного тестирования

## Исходные параметры
- Кластер: Minikube с 3 узлами
- Исходное количество подов: 3
- HPA: min=3, max=15, target=70% CPU

## Ход тестирования
- Запущено: 50 параллельных генераторов нагрузки
- Длительность: 10 минут
- Протокол: HTTP к LoadBalancer (127.0.0.1:80)

## Результаты масштабирования
- 00:00 - 3 pods (начальное состояние)
- 02:54 - 6 pods (первое масштабирование)  
- 03:53 - 9 pods (второе масштабирование)
- 04:20 - 15 pods (максимальное масштабирование)

## Метрики производительности
- Пиковая CPU утилизация: 164%
- Средняя CPU на под: 50-70m
- Память: 7Mi на под
- Распределение: по 5 подов на каждом узле

## Вывод
Кластер успешно обработал высокую нагрузку, HPA корректно масштабировал количество подов,
балансировщик нагрузки равномерно распределил трафик между всеми узлами.
